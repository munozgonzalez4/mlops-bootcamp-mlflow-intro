# mlops-bootcamp-mlflow-intro

This repository is my personal MLOps bootcamp notes and practical MLflow experiments. It is organized as an end-to-end exploration of MLflow concepts, model lifecycle management, experiment tracking, hyperparameter search, and model registry usage.

## What is in this repo

- `requirements.txt` — Python dependencies used across the notebooks.
- `src/getting_started.ipynb` — Basic MLflow tracking server check and a first experiment to verify local tracking.
- `src/first_ml_project/getting_started.ipynb` — End-to-end MLflow example using Logistic Regression on the Iris dataset with experiment runs, logging, model registry, inference, and validation.
- `src/house_price_prediction/main.ipynb` — California housing regression example that uses `RandomForestRegressor`, `GridSearchCV`, MLflow tracking, model logging, and `skops` serialization.
- `src/ann/starter.ipynb` — ANN model workflow for wine quality prediction with Keras, Hyperopt hyperparameter optimization, nested MLflow runs, model registry guidance, and deployment-oriented notes.
- `src/ann/mlruns/` and `src/house_price_prediction/mlartifacts/` — stored artifacts and saved model outputs generated by the notebooks.

## Learning goals and key topics

- MLflow experiment tracking and metrics logging.
- Using `mlflow.set_tracking_uri(...)` to connect code to a local server.
- Creating experiments and runs in MLflow.
- Logging parameters, metrics, tags, and model artifacts.
- Capturing model signatures with `infer_signature`.
- Registering models in the MLflow Model Registry.
- Comparing model versions and selecting the best run.
- Using MLflow with scikit-learn, TensorFlow/Keras, and `skops` model serialization.
- Hyperparameter tuning using `GridSearchCV` and Hyperopt.
- Validating saved models and testing inference from artifact storage.
- Principles of model lifecycle management and deployment readiness.

## Setup

1. Create or activate your Python environment.
   - Example using conda:
     ```bash
     conda create -p venv python=3.10 -y
     conda activate venv/
     pip install -r requirements.txt
     ```
2. Start the MLflow tracking UI:
   ```bash
   mlflow ui
   ```
3. Open the UI in your browser at:
   - `http://127.0.0.1:5000`
4. Run the notebooks from `src/` and follow the notebook cells.

## Important notes

- The notebooks assume a local MLflow tracking server running on `http://127.0.0.1:5000`.
- `src/house_price_prediction/main.ipynb` includes logic for both local file store and remote tracking server behavior.
- `src/ann/starter.ipynb` is focused on comparing MLflow runs and preparing a model for deployment, not only training.
- `src/first_ml_project/getting_started.ipynb` includes sections on model registry tracking and inferencing from registered models.

## Practical takeaways

- Use `mlflow.log_params`, `mlflow.log_metric`, and `mlflow.log_artifact` to keep experiments reproducible and auditable.
- Track every run for better comparison and eventual model promotion.
- Register the best-performing model versions in MLflow to move from experimentation to production readiness.
- Capture model signatures so inference is consistent and the model can be served or packaged reliably.
- Keep environment dependencies in `requirements.txt` and rebuild the `venv` if needed.
